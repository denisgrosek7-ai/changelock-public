# Evaluation Guide

This public repository is meant to help buyers and technical reviewers understand product shape without exposing private implementation code.

## Recommended reading order

1. [README](../README.md)
2. [Product Overview](product-overview.md)
3. [Architecture](architecture.md)
4. [Security Model](security-model.md)
5. [Deployment Modes](deployment-modes.md)
6. [Cross-Cluster Model](cross-cluster-model.md)

## What this public repo is good for

- product positioning review
- architecture review
- solution-fit conversations
- buyer education
- partner orientation

## What still requires controlled access

- implementation review
- source-level due diligence
- deployment packaging review
- runtime behavior verification in detail
- internal operational hardening details

## Practical note

If a reviewer needs to validate how the product is actually implemented, that should happen through the separate private repository or a controlled technical review process.
