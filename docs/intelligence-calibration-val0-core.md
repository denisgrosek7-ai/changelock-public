# Intelligence Calibration Val 0 Core

Točka 5 / Val 0 establishes the fail-closed foundation for the Intelligence Calibration Program.

It defines calibration discipline only. Intelligence remains advisory and projection-only, never canonical truth.

Val 0 covers:

- calibration dataset discipline
- confidence and evidence-class model
- intelligence output lifecycle
- reachability evidence taxonomy
- VEX candidate lifecycle
- feedback classification
- learning mode guardrails
- suppression safety
- federated signal boundary
- calibration provenance chain
- freshness and expiry handling
- calibration metrics definitions
- false-positive and false-negative discipline
- rollback and undo contract

Key guardrails:

- datasets, confidence, reachability, VEX candidates, feedback, suppression, federated hints, and metrics must not become canonical truth
- feedback, suppression, learning mode, and federated signals do not gain mutation or approval authority in Val 0
- stale, expired, unknown, and unsupported intelligence states remain explicit and bounded
- calibration changes remain traceable, reviewed, and rollback-aware

Val 0 does not complete Točka 5. Later waves still need:

- Val A: bounded reachability and VEX calibration engines
- Val B: behavioral baseline and anomaly baseline engines
- Val C: reviewed feedback and federated tuning flows
- Val D: simulation, scoreboard, and final calibration gate
- Val E: integrated closure
