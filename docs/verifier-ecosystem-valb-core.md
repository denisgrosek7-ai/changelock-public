# Verifier Ecosystem Val B Core

Točka 7 / Val B implements compatibility, diagnostics, and conformance only.

Val B adds:

- a verifier compatibility matrix
- schema and proof-type compatibility evaluation
- mixed-version diagnostics
- deterministic diagnostic precedence
- bounded fixture descriptors
- verifier conformance cases and conformance suite evaluation
- verification output classes

Compatibility matrix output remains advisory and version-bound. It does not certify a verifier or create canonical truth.

Conformance suites remain bounded behavioral checks against declared fixtures and cases only. They do not certify a verifier, approve deployments, or create marketplace ratings.

Fixture descriptors are test and conformance descriptors only. They are not fake production evidence and do not mutate the canonical execution, audit, or evidence spine.

Diagnostic precedence remains deterministic and fail-closed. Unsupported, stale, revoked, superseded, malformed, incomplete, or scope-breaking artifacts remain explicitly non-verified.

Val B does not implement:

- public verifier hub
- partner or auditor portal
- third-party publisher profile
- final Točka 7 closure
- point_7_pass

Verifier compatibility, diagnostics, conformance, and output classes remain advisory projections over the canonical execution, audit, and evidence spine.

Točka 7 remains `not_complete` until Val E integrated closure.
